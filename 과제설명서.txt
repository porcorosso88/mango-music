가. 웹페이지 개요
1. 음악 감상 페이지를 만들어 보고 싶었습니다.
2. 사이드 메뉴보다는 고정형 상단 내비게이션 메뉴를 이용했습니다.
최소한의 클릭으로 주요 콘텐츠를 보는 것을 목표로 했기 때문에
한 페이지에 주요 내용을 다 담으려고 했습니다. 사이드바 메뉴를 클릭해서 
들어가는 것이 아니라 스크롤만 내리면 주요 서비스를 이용 혹은 감상할 수 있도록 
구상했습니다.  

나. 구현한 내용
1. 3초마다 변화하는 앨범 자켓 사진
  첫 화면에 앨범 자켓 사진이 바뀌는 것을 보여주어 
  사용자의 흥미를 이끌어내려고 했습니다.
  
2. 앨범자켓에 링크 설정
  마음에 드는 자켓 사진을 클릭하면 
  해당 앨범을 소개하는 영역으로 이동하도록 했습니다.

3. 앨범 소개 사진에 1분 미리 듣기 버튼 구현
  앨범 소개 영역에 앨범 자켓 사진, 소개글 보면서 
  1분 간 음원을 감상할 수 있도록 만들었습니다.

4. 로그인 페이지
  내비게이션 바, 앨범 소개 영역에 있는 로그인 버튼을 누르면
  로그인 화면으로 이동하도록 했습니다. 그리고 아이디, 비밀번호를
  잘못 입력하면 아이디, 비밀번호를 알려주는 경고창이 뜨도록 했습니다.
  실서비스가 아니라 로그인 코드가 정상적으로 작동하는지 확인하기 위한 용도입니다.
  (아이디:abc@naver.com 비밀번호:abc)

5. 로그인한 사용자를 환영하는 문구와 로그아웃 버튼
  로그인에 성공하면 다시 메인 페이지로 이동합니다. 그리고 화면 상단에
  가입, 로그인 메뉴가 사라지고 환영 메시지와 로그아웃 버튼이 나타납니다.
  로그아웃 버튼을 누르면 다시 가입, 로그인 메뉴가 뜨게 했습니다.
  일단 이 정도로 로그인, 로그아웃 기능을 구현해봤습니다.

6. 검색창에서 가수 이름 자동완성 기능
  datalist 태그를 이용해서 자동완성 기능을 구현했습니다. 
  일단 3명의 가수가 데이터베이스에 있다고 가정했습니다. 
  예를 들어, '박효신'을 검색창에 입력하고 검색 버튼을 누르면
  박효신의 앨범 소개 영역으로 이동하는 것으로 검색 기능을 구현해봤습니다.


다. 구현하지 못한 내용
아래 내용은 CSS, 부트스트랩에 대한 이해가 부족하거나 
단기간에 배워서 구현하기에는 어려워서 포기한 내용들입니다.

1. 라디오 스트리밍 기능
2. 가수별 앨범목록, 수록곡 검색하는 기능
3. 노래 가사와 전곡 감상하는 페이지 
4. 반응형 웹페이지 미완성
  a. 미리 듣기 버튼 위치 불안정
  b. 모바일 화면에서 토글 메뉴 색깔이 내비게이션 바 색깔과 동일한 문제


라. 느낀 점
부트스트랩을 처음 써봤기 때문에 예제 동영상과 해외 개발자들이 만든 템플릿
소스를 참조하면서 만들었습니다. 하지만 아직 CSS에 대한 이해가 부족하고
웹페이지 디자인을 깊이 있게 공부한 적이 없어서 원하는 대로 만들지 못해서 아쉽습니다.

HTML, CSS는 자바스크립트보다 쉬울 것이라 생각했는데 막상 웹페이지를 만들어보니
어느 것 하나 안 중요한 것이 없었습니다. 이제부터라도 자바스크립트 기초를 계속 다지면서
HTML, CSS에 대한 공부도 하겠습니다.
